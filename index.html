LINE 群組綁定流程

系統使用者 → LINE Login → Token 產生 → 群組綁定

👤 使用者操作
🖥️ 系統 / 後端
💬 LINE App
── PHASE 1：系統登入 ──
1登入系統
輸入帳號密碼
2驗證身份
Session 存入
userinfo.id
── PHASE 2：LINE Login 身份綁定 ──
3進入「通知設定」
點擊
LINE 帳號綁定
4產生 state
存入 Session
導向 LINE OAuth
5LINE 授權頁
使用者點擊「同意」
profile + email scope
↓ LINE 回傳
authorization code
到 Callback URL
🔄 LINE → 後端 Callback：帶入 code + state
6驗證 state
用 code 換 id_token
解析取得
userId + email
7更新 userinfo
SET line_user_id=Uxxx
SET line_email=xxx
8顯示綁定成功
LINE 帳號已連結
── PHASE 3：產生群組綁定 Token ──
9點擊
「產生邀請碼」
10產生 Token
ABC12345
有效期 30 分鐘
11寫入
line_register_token
token / userinfo_id / expired_at
12畫面顯示
ABC12345
複製 token
── PHASE 4：群組綁定（可重複 N 次）──
13將 Bot 加入
LINE 群組 A / B / C
14Webhook 收到
join 事件
groupId 暫時記錄
15在群組內輸入
/register ABC12345
16Webhook 收到
message 事件
groupId + token
🔄 groupId + token 送到後端處理
17查詢 token
驗證未過期
取得 userinfo_id
是否已綁定？
existsByUserinfoIdAndGroupId
❌ 已綁定 → 回覆提示 ✅ 未綁定 → 繼續
18寫入
line_group_binding
userinfo_id / group_id / enabled=true
📲 Bot 回覆「✅ 群組綁定成功！」
19群組收到確認訊息
✅ 綁定完成
── 重複步驟 13–19 綁定群組 B、C ... ──
🔁 同一個 Token（ABC12345）在有效期內可重複使用，每個群組各執行一次 /register 即可綁定多個群組
── PHASE 5：綁定完成，可發送通知 ──
20系統顯示
已綁定群組清單
可勾選啟用 / 停用
21查詢
line_group_binding
where userinfo_id=X
and enabled=true
22推送通知
到所有啟用群組
push → group_id A,B,C
23群組 A / B / C
收到事件通知 🔔

📋 資料表對應關係

資料表 用途 關鍵欄位
userinfo 系統使用者，加入 LINE 綁定欄位 id, line_user_id, line_email
line_register_token 群組綁定邀請碼，有效期內可重複使用 token, userinfo_id, expired_at
line_group_binding userinfo.id ↔ groupId 對應，可勾選啟用 userinfo_id, group_id, enabled
